================================================================================
  SMART HOTEL INTELLIGENCE SYSTEM
  A Data Science Product for Hotel Booking Analytics
================================================================================

PROJECT OBJECTIVE
-----------------
Hotels lose millions annually due to unexpected booking cancellations and
inaccurate revenue forecasting. This system gives hotel managers predictive
intelligence BEFORE problems happen — with no data science knowledge required.

The system provides two core AI-powered features:
  1. Cancellation Risk Predictor — scores any booking 0-100% risk with
     actionable manager recommendations (e.g., request deposit, send reminder)
  2. Revenue Estimator — predicts the expected nightly room rate (ADR) for
     any booking, benchmarked against the hotel's historical average
  3. Analytics Dashboard — data-driven insights across 119,390 real bookings


DATASET
-------
Source : Hotel Booking Demand dataset (open-source, real-world data)
         Antonio, Almeida & Nunes (2019) — hotel_booking.csv
Records: 119,390 hotel bookings (City Hotel + Resort Hotel)
Features: 36 columns including lead time, guest type, deposit, meal plan, ADR


TOOLS & TECHNOLOGIES
---------------------
Language   : Python 3.9+
Framework  : Streamlit  (web application)
ML Library : Scikit-learn (Random Forest Classifier + Regressor)
Visualisation : Plotly
Data Processing: Pandas, NumPy

INSTALLATION STEPS
------------------
Step 1: Ensure Python 3.9 or higher is installed.
        Check with:  python --version

Step 2: (Recommended) Create and activate a virtual environment:
        python -m venv venv
        venv\Scripts\activate          (Windows)
        source venv/bin/activate       (Mac/Linux)

Step 3: Install required packages:
        pip install -r requirements.txt

Step 4: Make sure hotel_booking.csv is in the project root folder.


TRAINING THE MODELS (Required Once)
--------------------------------------
Before launching the app, train the ML models by running:

        python train_models.py

This will:
  - Load and preprocess hotel_booking.csv
  - Train a Random Forest Classifier (cancellation prediction)
  - Train a Random Forest Regressor (revenue/ADR prediction)
  - Save all models and metadata to the model/ folder
  - Print accuracy and R² scores on test data

Training takes approximately 2-4 minutes depending on your hardware.
Once models are saved, you do NOT need to retrain unless data changes.


RUNNING THE APPLICATION
-----------------------
        streamlit run app.py

The app will open automatically in your web browser at:
        http://localhost:8501

If it does not open automatically, copy the URL from the terminal.


APPLICATION TABS & USAGE
--------------------------
Tab 1 — Overview
  Describes the business problem, model architecture, and performance metrics.

Tab 2 — Cancellation Risk Predictor
  INPUT : Booking details (hotel type, lead time, guest info, deposit type, etc.)
  OUTPUT: Cancellation probability (%) with colour-coded risk level and
          specific recommended actions for hotel manager.

Tab 3 — Revenue Estimator
  INPUT : Booking details (same fields, excluding room rate)
  OUTPUT: Predicted nightly room rate (ADR in €), total stay revenue,
          comparison against hotel historical average, pricing advice.

Tab 4 — Analytics Dashboard
  No inputs required. Displays interactive charts including:
  - Monthly booking volume (colour-coded by cancellation rate)
  - Cancellation rate by market segment
  - Average room rate trends by month & hotel type
  - Impact of booking lead time on cancellation risk
  - Cancellation rate by deposit type
  - AI feature importance (top risk drivers)


INPUTS & OUTPUTS SUMMARY
--------------------------
Cancellation Predictor inputs:
  hotel type, arrival month, lead time (days), deposit type, market segment,
  customer type, weekend nights, week nights, meal plan, room rate,
  adults, children, babies, special requests, parking, repeat guest flag,
  previous cancellations, previous completed stays, booking changes,
  days on waiting list

Revenue Estimator inputs:
  hotel type, arrival month, lead time, room type, market segment,
  deposit type, weekend nights, week nights, meal plan, customer type,
  adults, children, babies, special requests, parking, repeat guest flag,
  booking changes

Outputs:
  - Cancellation risk: percentage + risk level (Low / Medium / High)
                       + revenue at risk (€) + manager action list
  - Revenue estimate:  predicted ADR (€/night) + total stay revenue (€)
                       + % difference from hotel average + pricing advice
